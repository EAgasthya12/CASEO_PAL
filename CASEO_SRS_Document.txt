SOFTWARE REQUIREMENTS SPECIFICATION (SRS)
PROJET: CASEO - AI-POWERED EMAIL INTELLIGENCE PLATFORM
VERSION: 1.0
=====================================================

1. INTRODUCTION
---------------
1.1 Purpose
The purpose of this document is to provide a detailed overview of the software requirements for the CASEO platform. This document will describe the functional and non-functional requirements of the system.

1.2 Scope
CASEO is an intelligent email assistant that integrates with Gmail to provide automated categorization, urgency detection, and summarization using Large Language Models (LLMs). The goal is to reduce "email fatigue" by prioritizing critical communications.

1.3 Definitions and Abbreviations
* SRS: Software Requirements Specification
* LLM: Large Language Model (e.g., Gemini 1.5 Flash)
* OAuth: Open Authorization (Standard for access delegation)
* JWT: JSON Web Token


2. OVERALL DESCRIPTION
-----------------------
2.1 Product Perspective
CASEO is a standalone web application comprising a Node.js backend, a React.js frontend, and a Python-based intelligence layer. It interacts with external APIs including Google Gmail API and Google Gemini API.

2.2 Product Functions
* Secure user authentication via Google OAuth.
* Background synchronization of email threads from Gmail.
* AI-driven classification into categories (Academic, Job, Finance, etc.).
* Automated extraction of deadlines and urgency levels.
* One-click summarization of long email content.
* Dashboard for managing "Priority" (Urgent/Important) communications.

2.3 User Classes and Characteristics
* End User: Any individual with a Gmail account who seeks better organization of their inbox.

2.4 Operating Environment
* Server: Node.js 18+, Python 3.9+, MongoDB.
* Client: Modern Web Browsers (Chrome, Firefox, Safari, Edge).
* Mobile: Android (via Capacitor wrapper).


3. SYSTEM FEATURES
------------------
3.1 Feature: Gmail Integration
* Requirement: The system must fetch new email threads using the Gmail API.
* Output: Email content stored in local MongoDB for fast retrieval.

3.2 Feature: Intelligent Classification
* Requirement: Use Gemini LLM to categorize emails based on "true intention."
* Logic: Categorize into Academic, Job, Finance, Internship, Event, Newsletter, or Personal.

3.3 Feature: Priority Filtering
* Requirement: Automatically identify "Critical" or "High" urgency items (e.g., OTPs, interview invites, near-deadline assignments).
* Visualization: Display these items in a dedicated "Priority" tab.

3.4 Feature: Email Summarization
* Requirement: Provide a 3-4 sentence professional summary of any selected email thread.
* Output: Summary text with actionable next steps.


4. EXTERNAL INTERFACE REQUIREMENTS
-----------------------------------
4.1 User Interfaces
* A responsive dashboard showing email lists with urgency tags.
* A sidebar for navigation between Inbox, Priority, Sent, and Spam.
* A detailed view for reading summaries and changing categories.

4.2 Software Interfaces
* Google Gmail API v1: To fetch and manage emails.
* Google Generative AI (Gemini Flash 1.5): For NLP and processing.
* MongoDB Atlas: For database storage.

4.3 Communication Interfaces
* The app uses HTTPS for all external API calls.
* The Frontend communicates with the Backend via RESTful JSON APIs.


5. NON-FUNCTIONAL REQUIREMENTS
------------------------------
5.1 Performance
The system should categorize a batch of 10 emails in under 5 seconds. The Dashboard must render stored emails in under 300ms.

5.2 Security
* No user passwords stored (OAuth only).
* Database queries restricted by User-ID (Authorization).
* Input sanitization for all AI-provided data.

5.3 Reliability
The system must fall back to local keyword-based classification if the LLM API is unreachable.

5.4 Availability
Target availability is 99.5% for the web interface.


6. OTHER REQUIREMENTS
----------------------
* The application must support mobile viewports via responsive CSS.
* The system should handle up to 500 emails per user sync session.
